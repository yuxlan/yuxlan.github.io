angular，react，vue

2017-09-08

react教程
vue教程
angular教程

数据流

1.angular 使用数据双向绑定：界面的操作能实时反映到数据，数据的变更能实时展现到界面
2.vue 也支持数据双向绑定，默认为单向绑定，数据从父组件单向传给子组件
3.react 函数式编程和单向数据流
4.在大型应用中使用单向绑定让数据流易于理解【vue,react】

数据渲染

1.angular：html模板将会被浏览器解析到DOM中，DOM结构成为angular编译器的输入；angular将会遍历DOM模板，生成相应的NG指令，所有的指令都负责针对view（ng-modal）来设置数据绑定；NG指令是在DOM加载完成之后才开始起作用
2.react：渲染建立在虚拟DOM上，当状态发生变化时，react重新渲染虚拟DOM，比较计算之后再传给真实DOM；虚拟DOM提供了函数式的方法描述视图，每次更新都会重新渲染整个应用，从定义上保证了视图与数据的同步
3.在超大量数据的首屏渲染速度上，react有一定优势，支持服务端渲染和原生渲染，angular也支持服务端渲染和原生渲染

模块化与组件化

1.angular使用依赖注入解决模块之间的依赖问题，使用ES6的module来定义模块，也考虑了动态加载的需求【我们使用的angular2】
2.vue中指令和组件分得更清晰，指令只封装DOM操作，组件代表一个自给自足的独立单元（有自己的视图和数据逻辑）
3.react应用是构建在react组件上的（组件化编程）

语法和代码风格

1.react，vue，angular都支持ES6
2.react以JavaScript为中心【声明式设计（采用声明范式，可以轻松描述应用），高效（通过对DOM的模拟，最小限度地减少与DOM的交互），灵活（可以和已知的库或框架更好地配合）】
3.angular2以html为中心【良好的应用程序结构，双向数据绑定，指令，html模板，可嵌入、注入和测试】
4.vue通过简单的API（应用程序编程接口）提供高效的数据绑定和灵活的组件系统【轻量级的框架，支持单向数据也支持双向数据绑定，指令，插件化】

比较

1.react适合开发轻量级单一组件，是最灵活的视图渲染框架
2.react更小并且只涵盖视图组件的操作，缺少向html的扩展机制是react的不足
3.react速度快（在UI渲染过程中，通过在虚拟DOM中的微操来实现对实际DOM的局部更新），跨浏览器兼容，模块化，单向数据流，同构纯粹的JavaScript，兼容性更好
3.angular2适合时间处理逻辑多，UI渲染复杂的
4.angular2更稳定、可扩展且更加完善
5.vue更简单（官方文档清晰，比angular简单易学）；快速（异步批处理方式更新DOM）；组合（用可复用的组件组合应用程序）；紧凑；强大（表达式&无需声明依赖的可推到属性）；对模块友好；不支持IE8及以下

react

var User = function (name, status) {
this.name = name;
this.status = status;
}

User.prototype={
    save:function(name,status){
        //save to db
    }
}
var user = new User();

var RegisterComponent = React.createClass({
    displayName: "RegisterComponent",
    getInitialState: function () {
        return user;
    },
    onFirstInputChange: function () {        
        this.setState({name: React.findDOMNode(this.refs.fistInput).value});
    },
    onSecInputChange:function(){       
        this.setState({status: React.findDOMNode(this.refs.secInput).value});
    },
    save:function(){
        //只有不更改input的时候这个返回true,setState之后就成false
        console.log(this.state===user)
        //所以使用this.state吧
        user.save(this.state.name,this.state.status);
    },
    render: function () {     
        return (
         <div>
            <div>name:   <input type="text" onChange={this.onFirstInputChange}  ref="fistInput" value={this.state.name} />   </div>
            <div>status: <input type="text"  onChange={this.onSecInputChange} ref="secInput" value={this.state.status}  /> </div>
            <button onClick={this.save}>save</button>


        </div>
        );
    }
});


React.render(<RegisterComponent />, document.getElementById("RegisterComponentExample"));
